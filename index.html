jenkins

[toc]

环境

java version “21.0.1” 2023-10-17 LTS (jenkins编译要求java11、17、21)

Apache Maven 3.9.6

Git 2.43.0

Jenkins安装(msi)

Jenkins_Download

  1. 以本地系统服务安装

  1. 修改jenkins端口，8080会和tomcat冲突(这里是8088)

  2. 选择你的jdk路径(版本必须是java11、17、21)

  3. 其余步骤”Next”即可。

  4. 在浏览器中访问”localhost:8088”，并进入到对应路径获取密码。

  5. 以笔记本形式打开，并将其中的密码复制到输入框中并点继续。

  6. 安装推荐插件即可，安装失败也可以进Jenkins的plugins中安装。

  7. 创建用户(也可以直接用admin)

  8. 完成安装。

Jenkins插件安装

安装maven插件、git插件、PublishOverSSH、Build Authorization Token RootVersion

Maven Integration

Git (正常来讲前面没报错就已经是安装好的状态)

Build Authorization Token Root (用于webHook免密post)

PublishOverSSH

  1. 勾选对应的插件前方的方框，点击安装。

  1. 安装好重启jenkins (Manage Jenkins -> Reload Configuration From Disk)

maven插件配置

  1. 进入设置界面

  2. 拉到页面最底端

  3. 输入你本地的mvn路径，也可以直接用自动安装(这里用本地的)

构建一个项目

  1. 新建item

  2. 以Manven项目构建(需要刚刚的maven插件)

  3. 添加仓库链接

  4. 添加账号

  5. 选择部署的仓库分支

  6. 这时候就可以保存尝试Build了。

  7. 进入控制台查看是否构建成功

  1. 在这里可以看到构建完jar包的位置。

  2. 可以尝试部署

构建并部署

这里部署在另外一台CentOS7的虚拟机(192.168.220.100)上，已经配置好对应的数据库和jre环境。

  1. 进入Manage Jenkins -> System，找到SSH Servers。

  2. 新增一个SSH链接，填写完可以点击右下角的”Test Configuration”测试连接。

  3. 进入对应item的界面，找到Post Steps，选择”Send FIles or execute commands over SSH”(要安装publish over ssh 插件才有).

    ​

  4. 如果服务器正确连接且配置正确的话，这时候保存并”Build Now”就可以自动编译部署到服务器了！

图床(Typora+smms+PicGo)

markdown文档一般都会将文档中的图片保存到本地，如果直接将.md文档发送给别人或者发布至网络，一般图片都会加载不出来。所以需要将我们的图片放到存放图片的服务器(图床)来供网络访问。

这里使用的md编辑器为typora配合PicGo使用，图床smms有5G的免费空间。

  1. 进入smms注册账号，打不开的话点这个国内备用。

  2. 注册完成后进入dashboard。

  1. 获取对应的API token，建议保存到记事本中，等会要用。

  1. 下载PicGo(可以直接在typora里下载或者去github的release中下载)

  1. 将PicGo路径设置为你安装的PicGo路径中的PicGo.exe，修改插入图片时的的操作。

  1. 打开PicGo并进行图床设置(用什么图床选什么)

  1. 将刚刚的token填入其中，备用上传域名填国内可以访问的smms.app

  1. 至此便可以尝试插入图片测试是否成功了!
