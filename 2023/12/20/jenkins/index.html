jenkins

[toc]

环境

java version “21.0.1” 2023-10-17 LTS (jenkins编译要求java11、17、21)

Apache Maven 3.9.6

Git 2.43.0

Jenkins安装(msi)

Jenkins_Download

  1. 以本地系统服务安装

  1. 修改jenkins端口，8080会和tomcat冲突(这里是8088)

  2. 选择你的jdk路径(版本必须是java11、17、21)

  3. 其余步骤”Next”即可。

  4. 在浏览器中访问”localhost:8088”，并进入到对应路径获取密码。

  5. 以笔记本形式打开，并将其中的密码复制到输入框中并点继续。

  6. 安装推荐插件即可，安装失败也可以进Jenkins的plugins中安装。

  7. 创建用户(也可以直接用admin)

  8. 完成安装。

Jenkins插件安装

安装maven插件、git插件、PublishOverSSH、Build Authorization Token

Maven Integration

Git (正常来讲前面没报错就已经是安装好的状态)

Build Authorization Token Root (用于webHook免密post)

PublishOverSSH

  1. 勾选对应的插件前方的方框，点击安装。

  1. 安装好重启jenkins (Manage Jenkins -> Reload Configuration From Disk)

maven插件配置

  1. 进入设置界面

  2. 拉到页面最底端

  3. 输入你本地的mvn路径，也可以直接用自动安装(这里用本地的)

构建一个项目

  1. 新建item

  2. 以Manven项目构建(需要刚刚的maven插件)

  3. 添加仓库链接

  4. 添加账号

  5. 选择部署的仓库分支

  6. 这时候就可以保存尝试Build了。

  7. 进入控制台查看是否构建成功

  1. 在这里可以看到构建完jar包的位置。

  2. 可以尝试部署

构建并部署

这里将把JAR包部署在另外一台CentOS7的虚拟机(192.168.220.100)上，服务器上已经配置好对应的数据库和jre环境。

  1. 进入Manage Jenkins -> System，找到SSH Servers。

  2. 新增一个SSH链接，填写完可以点击右下角的”Test Configuration”测试连接。

  3. 进入对应item的界面，找到Post Steps，选择”Send FIles or execute commands over SSH”(要安装publish over ssh 插件才有).

    ​

  4. 如果服务器正确连接且配置正确的话，这时候保存并”Build Now”就可以自动编译部署到服务器了！

  5. 每次部署都会运行一个jar包，会导致上一个进程一直运行并且端口被占用，可以通过”Pre Steps”在发送文件前将上一个进程kill掉和清理环境(这里仅kill掉程序,需要更多操作可以通过shell实现)

  6. 保存并再次执行”Build Now”

提交代码时自动重新编译打包部署(WebHook)

这里通过gitee的webhook，在向仓库提交代码后，gitee会向jenkins发送一个post，jenkins在接收到post后就会自动build，从而实现代码提交时的自动编译打包部署。

如果jenkins部署在内网，则需要代理或内网穿透了

  1. 在item设置中开启”触发远程构建”

    1. 进入gitee仓库的管理 -> WebHooks 新增一个webHook

    2. 填写WebHook的信息

      这里填"/buildByToken/build?job={项目名}&token={Jenkins中的身份验证令牌}"而不是jenkins推荐的"JENKINS_URL‘“job/book_manager_system/build?token=TOKEN_NAME” 或者 “/buildWithParameters?token=TOKEN_NAME"是因为jenkins默认的需要身份验证，而buildByToken是由插件”Build Authorization Token”提供，在post时不需要验证。

    3. 点击测试按钮看是否能通过测试，如果通过测试则jenkins会自动Build一遍。

    4. 至此如果配置正确，已经可以通过代码上传至仓库时，自动编译打包部署了！

构建后进行邮件通知

在项目进行构建后可以通过发邮件的方式通知其他人，发邮件的用户需要开通SMTP功能，这里以QQmail为例。

  1. 开通SMTP （找到设置 -> 账号 -> POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务）

    1. 在jenkins中设置发信账号 (Manage Jenkins -> System)

    2. 配置Extended E-mail Notification

    3. 配置系统全局的邮件通知

    4. 配置item的发信配置 (configure -> 构建后操作 -> 添加构建后操作 -> Editable Email Notification)

    5. 可以在advanced Setting中设置相关的Trigger

    6. 至此已经完成构建后发送邮件！
